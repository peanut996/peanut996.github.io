19.解释ORM思想的主要内容

ORM(Object Rational Mapping)的思想：将关系数据库中表中的记录映射成为对象，以对象的形式展现，程序员可以把对数据库的操作转化为对对象的操作。

20.解释IOC思想的主要内容

控制反转(IoC=Inversion of Control)，控制反转(或依赖注入) ，用白话来讲，就是由容器控制程序之间的（依赖）关系，而非传统实现中，由程序代码直接操控。这也就是所谓“控制反转”的概念所在：(依赖)控制权由应用代码中转到了外部容器，控制权的转移，是所谓反转。满足软件设计原则中的DI(依赖倒置)原则。

21.解释AOP思想的主要内容

AOP即Aspect-Oriented Programming的缩写，中文意思是面向切面（或方面）编程。AOP实际上是一种编程思想，可以通过预编译方式和运行期动态代理实现在不修改源代码的情况下给程序动态统一添加功能的一种思想。

22.Java中POJO类的主要特点

POJO是Plain OrdinaryJava Object的缩写，但是它通指没有使用Entity Beans的普通java对象，可以把POJO作为支持业务逻辑的协助类。
POJO实质上可以理解为简单的实体类，顾名思义POJO类的作用是方便程序员使用数据库中的数据表，对于广大的程序员，可以很方便的将POJO类当做对象来进行使用，当然也是可以方便的调用其get,set方法。POJO类也给我们在struts框架中的配置带来了很大的方便。区别于Bean类。

第一次折腾黑苹果

Posted on 2019-10-07
Symbols count in article: 930 Reading time ≈ 1 mins.

前言

国庆节提前了一天回来，于是多了一天的空闲时间，于是便打算奖励自己一天折腾黑苹果，其实很久之前就很想搞这个了，一直没时间。(其实我以为它很耗时间)

准备

看了一下各种论坛和博客，大致的过程就是找到合适的Clover配置文件，然后能成功引导安装Mac OS，之后再配置各种驱动(kext)，因为以前装系统，所以基本功还算扎实。上网搜了一下光影精灵3的配置文件，然后发现了可能可以用的文件，然后这里下载的是果黑小兵的部落阁提供的Mojave镜像。然后需要准备的工具:

  • 8GB以上的U盘

  • 以太网

  • Tranmac

  • DiskGenius

开始

U盘安装

首先就是先在电脑上分出一块磁盘，看个人需求吧，做主力机的话就大一点的，然后用Transmac安装启动盘，一般安装后会提示你windows无法打开需要格式化，直接取消即可。然后打开DiskGenius查看U盘会有一个EFI分区，把里面的Clover文件夹替换成自己找到的文件，关机，Bios设置成第一启动，进入Clover选择Boot install from Mac OS进入安装程序，接下来就是安装过程了。

安装过程

在一段时间加载之后就会进入Mac OS常用工具了，因为MacOS安装需要苹果专用的分区表，我们必须格式化磁盘，选择磁盘工具，然后把刚刚那块空白的磁盘抹除为”MAC日志文件”格式，需要注意的是需要保证该块磁盘所在的硬盘EFI空间大于200M，否则会出现抹除失败的情况。然后进入安装程序，选择刚刚那块磁盘，然后就会开始安装了，期间可能会重启几次，再进入安装程序都是正常的现象，最后等待大概半个小时左右，就可以进入系统了，但是还没结束，才是刚开始，接下来就是漫长的配置驱动过程了。

转移引导

直到目前为止采用的都是U盘引导，接下来把引导文件放到硬盘里，因为主系统是Windows，采用的是UEFI+GPT已经有了EFI分区，用DiskGenius把Clover移动到\\efi文件夹下即可。

遇到问题

没错，我总是能碰到这么奇怪的问题，就在转移引导的时候，使用硬盘引导的时候CPU居然直接挂掉了，但是在U盘引导的时候就没有这样的问题，最后用了大佬给的Clover就可以正常用了，然后他的驱动也是配置好的，就相当于省了不少步骤，有点偷懒，嘿嘿。

前言

众所周知，祖国母亲70岁生日，防火墙就疯狂工作了，导致我的ssr几乎5分钟炸掉一次，没办法看了下其实v2ray的安全性还可以，而且上次申请专利的人也撤回了申请，就直接选了安全性比较不错的一种方法——伪装成HTTPS流量，接下来就简单说下需要准备的东西，也只是个简略的教程，具体细节Github以及官网也都有说。

准备

  • 一台能用的VPS
  • 一个正常的域名
    首先你的VPS大陆要能正常访问，推荐搬瓦工的Just my socks服务，挺省心的，然后系统选择安装Linux，最好选择Debian，然后安装好V2ray和Nginx，接下来在你的域名那里添加一个A记录指向VPS的IP地址。接下来你需要让自己的VPS拥有tls，也就是需要一个证书，我们选择免费的即可Let’s Encrypt,保证最后再使用Qualys SSL Labs’s SSL Server Test进行检测若评级为A则基本正常。最后一步就是配置Nginx让443端口进来的流量通过V2ray进行转发，具体的配置文件可参考新V2Ray白话文指南，这里也就不再啰嗦了。

问题

其实在配置的时候遇到了一个小问题，在streamsettings里面没有配置"sercuity":"auto"此项，导致一直报Websocket handshake failed 502 BadGateWay,记录下以免下次忘记。

踩坑

  • CheckBox只有在属性IsChecked 为false的状态下才能触发事件，并且一旦触发，属性立马改变而非执行完才改变。
  • MainWindow的属性为只读，不可修改，可以通过Maindow mainWindow = (MainWindow)System.Current.MainWindow; 获取一个实例。
  • Xaml和Xaml.cs两个文件编译后共同形成一个完整的Class这也是Partial修饰限定符的做法
  • bool?变量不可自动转换为bool变量，使用==false or true转换
  • git pull 与git fetch + git merge的区别
  • 删除非MainWindow的控件或者关闭时，最好新开线程，在该类的进程下关闭。

总结

算是从大一开始来自己独立完成的第一个项目了，其实这几天学到的东西比我这个学期上课学到的都要有用,比如说Git的常用操作，WPF的低级操作，还有最重要的SQL Sever的操作。后两天为了做一些美化工作，比如初始化等待窗口时用到了多线程和委托，感觉起来还是稍微有点复杂，但总算入门了。

How to do|思路

  • Supersu这是Android4.0时代就流传下来的老方法具体的内部实现方法好像是把需要root的进程以一种可执行文件的方式执行(这好像是所有root的本质)，具体可以看这个CSDN的链接但是只需要刷入卡刷包，开机就能获取su了。
  • Magisk简而言之，就是它修补了boot.img然后把/system下的东西全部定向到/systemless，这样的话它实际并没有动任何关于/system的文件，所以出了问题只需要禁用掉Magisk就行了。

Do it |操作

既然方法敲定那就开始吧，adb连接上adb reboot recovery然后一个倒地的机器人显示”无命令“，adb reboot bootloader然后是一直黑屏显示，但是应该是进入了某种模式。既然没有rec第一种方法就可以暂时放弃了。然后选择Magisk Manager的方法，修补boot.img,然后选择fastboot boot booot.img一直显示<waiting for any device>，以为是驱动问题，在网上换了很多的驱动最后还是无解。那就只能选择最后一种方法，把boot.img塞到update.zip直接卡刷升级，当然最后还是失败了。就当本来打算放弃的时候，在Hisense的官方论坛发现了M30的root和Twrp，当时觉得在bootloader刷机有点不靠谱。按照教程跑了一遍，最后真的成功了，卡Logo的时候确实比较着急的，开机以后打开Magisk Manager一看，果然成功了。比较惊呆了的其实是竟然还有人给这个机型做Twrp，但是没刷机包有什么用吗 :(

Sum|总结

总体搞下来其实也没花多长时间，主要是fastboot和bootloader有点没有标注，耽误了不少时间。
